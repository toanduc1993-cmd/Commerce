🎯 Key Design Decisions

📊 Excel-First Layout

Tất cả bảng tracking đều mirror 1:1 cấu trúc Excel mẫu. Sticky columns, group headers, sub-headers, color coding để user quen Excel có thể dùng ngay không cần training.

🔧 Dynamic Header Detection

Mỗi project file có offset cột khác nhau (col 9 vs col 12 vs col 7). Parser scan header row tìm "Item/STT" rồi build column map động — không hardcode.

🔗 Linkage qua itemCode

BidAnalysis → BidQuoteItem.itemCode link với PrDetail.itemCode. Contract → ContractDetail cũng link tương tự. Tất cả modules đều có thể trace ngược về 1 PR item.

♻️ Upsert Logic

Re-upload cùng file không tạo duplicate. Match by (contractNo, prDetailId) hoặc (supplier, saleContract) để update existing thay vì create.

🏷 Auto-detect Project

Sheet name match pattern 25-XXX-X-XXX → tự detect projectCode. Fallback dùng prefix itemCode (I95, I90) → match project tương ứng.

🎨 Workflow 11 Steps

Mỗi PrDetail có thể auto-infer trạng thái workflow từ data: có contract → "Đã ký HĐ", có arrivedDate → "Hàng đang về", có inspection PASS → "Nghiệm thu", có handover → "Đã nhập kho".

Layer 1: Nginx
  · TLS 1.2/1.3 + HSTS preload
  · login limit 1r/s burst 5
  · api limit 20r/s
  · max body 35M

Layer 2: Helmet CSP
  · default-src 'self'
  · frame-ancestors 'none'
  · object-src 'none'
  · referrer same-origin

Layer 3: express-rate-limit
  · login 5 fail/15min/IP
  · API 200 req/min/IP
  · upload 20 file/10min/IP

Layer 4: Auth
  · bcrypt 12 rounds
  · JWT 64-byte hex secret
  · audit log mọi LOGIN
  · isActive flag disable user
